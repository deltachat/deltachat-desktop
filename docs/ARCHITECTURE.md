# Architecture

This document provides an overview of the software architecture of the Deltachat Desktop client

Since we use Electron you should be familiar with the basic architecture of Electron, mainly the distinction between a "backend" running in a NodeJS context on your OS and a "frontend" running in a separate BrowserWindow (Chrome). Both are separate processes (see [Process Model](https://www.electronjs.org/docs/latest/tutorial/process-model))

They communicate via [Inter Process Communication](https://www.electronjs.org/docs/latest/tutorial/ipc)

You can access the Deltachat Core API in the frontend via an [ApiService](../src/renderer/apiService.ts) which provides types for all available API calls (which we can not link here since it is auto generated by Core)

Deltachat-Desktop includes a binary of the [Deltachat Core](https://github.com/deltachat/deltachat-core-rust) which also provides an [RPC Server](https://github.com/deltachat/yerpc/tree/main/yerpc) which enables to access the Core Api via JSON RPC. There is also an auto generated [JSON RPC client](https://github.com/deltachat/deltachat-core-rust/tree/main/deltachat-jsonrpc).
